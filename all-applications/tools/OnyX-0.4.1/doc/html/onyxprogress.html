The Current Progress
Curious when a bug is going to be fixed? When will change take place? Want to know when to expect a new feature? Just curious on the current progress of the next version? Use the following table to answer all of these questions.

It is an up to date table of the ongoing project work on OnyX. Each row represents one change or new feature. The first column is the name of the change. The other columns are future version numbers.

If an item is marked with X then it is projected to be available in that corresponding version or version range.

If an item is marked with - then it has been completed.

For a better description select the name of the feature/change for more information (if available).

How accurate is this table? Well, everytime I complete something for the next version I update it. So I would say it is a pretty good representation on the current progress of OnyX.

Absolute/Suggested Window

    Where should the windows of your application appear when they are first mapped to the screen? Will you just suggest a place and allow the Window Manager to be able to select a better position? Or do you need absolute control with absolute placement? This will be a new property with the OnyXWindow object.

Compiled Pixmaps

    Compiled pixmaps are pixmaps that are compiled into the executable image of the program. The down side of this is that the executable is always larger and takes up more memory. The up side is that the application never has to worry about missing pictures. They are always available. This option will be incorperated into OnyX so that OnyXIcons and other objects can make use of an internal pool of pictures at will.

    Return To Table

Container Object Property

    The current version of OnyX only allows for OnyXWindows to contain (be the parents of) other objects. This would be an additional property set in the OnyX Object File (.oob). When set to true, OnyX will allow the placement of other object inside of it during design time. See also: Multiple Parent.

    Return To Table

Context Menu
    Context menus are floating popup menus that tend to appear when ever you select a mouse button on an object. For example, right clicking on a window might bring up a context menu that offers choices of: viewing its resources, minimize or maximizing it, closing the window, and so on. Each context menu is specific to the object that was selected.

    Return To Table

Fixed Size Windows
Fixed Size Windows allow for the spefication of maximum or minimum window sized. By making these the same it is possible to keep a window from being resized. (Assuming the window manager doesnt override the setting.)
    Return To Table
Multiple Parent
    The current version of OnyX only allows for OnyXWindows to be parents of objects. This feature will allow any object to be a parent if it has the Container Object Property set to true. See also: Container Object Property, Multiple Parent Delete, Multiple Parent Rename.

    Return To Table

Multiple Parent Delete
    When objects can themselves be parents deleting a parent requires deleting or moving all of its children, grandchildren, etc. This must be implemented for safe use of Multiple Parents. See also: Multiple Parent, Multiple Parent Rename.

    Return To Table

Multiple Parent Rename
    When objects can themselves be parents renaming an object requires updating any references to it found in its children, grandchildren, etc. This must be implemented for save use of Multiple Parents. See also: Multiple Parent, Multiple Parent Delete.

Save Window Positions
    Getting a little aggravated because the windows in OnyX keep popping up in the exact same place that you don't want? This option will allow you to place the windows where you want and they will stay there, even when you start OnyX next time. :)

Shrink Wrap Toolbox

    This feature will automatically size the toolbar to be the exact width and height needed to show the buttons. You will still be able to resize it to determine how many buttons per row you want, but there wont be that extra space on the edges after it redraws.
