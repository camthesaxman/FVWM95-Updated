Installing Onyx

OnyX requires the Xclass library, which can be downloaded from sourceforge.

Chances are that Xclass is already installed on your machine, as some Linux distributions install it by default. One way to check which version of Xclass is installed on your machine is through the xc-config utility. That is a small shell script that comes with the library and that can be used, among other things, to check the Xclass version installed on your machine. Just type the command:

xc-config --version
If you get a "command not found" message from the shell, then Xclass has not been installed yet on your machine. Otherwise the xc-config will respond with the Xclass version number. You should get 0.9.2 or a higher number. If you have an earlier version, I would strongly recommend you to upgrade to the latest one, since the current version of OnyX uses features only found in Xclass 0.9.2 and up.

If necessary, download the latest Xclass library and install it following the instructions supplied with the package.

Once you have checked that you have the correct version of Xclass installed, you can proceed with the OnyX installation. This is done with an interactive setup program which will guide you step-by-step through the installation process in a simple and intuitive way.

First, unpack the OnyX source tarball:

tar zxf OnyX-0.9.2.tar.gz (if you have GNU tar)
gunzip -c file.tar.gz | tar xvf - (otherwise)
Then cd to the OnyX directory and run the setup program:

cd OnyX-0.4.2
./setup

Next, the setup program proceeds to copy the executable file and the bunch of OnyX objects to the appropiate directory. By default, a directory is created in /usr/local, but you can change that to something else if you prefer. Many Linux systems nowadays no longer install third-party applications under /usr/local, but instead under /usr/opt, /usr/share or something like that. In this example, we install OnyX under /usr/share (note that you should specify the full directory name where OnyX files will reside, and not just the parent):

At this point OnyX should be ready for use. You can try it right now, by entering the command:
OnyX
You should see the splash screen appearing, followed by the main window sticking to the top part of your display.

One extra touch: go to the Tools menu and select Options..., in the dialog window that appears set your favorite editor by changing the Editor Command field. The default is to run pico under an xterm, which definitively does not give a very professional look. You can use, for example, the notepad application that comes with xclass for a more uniform look and feel. Or perhaps you prefer gnome's gnotepad. Or emacs... is up to you. Just don't forget to specify an xterm if your favorite editor is a text-mode console application. Finish by clicking the Save button, and you're done with the OnyX installation.
